# Vue d'ensemble

L'ingénierie de prompts est une tâche de conception itérative. Les domaines de conception itérative plus formalisés (par exemple, la conception de logos) disposent d'outils bien établis qui permettent aux concepteurs d'être plus efficaces. Des outils similaires existent dans l'espace de l'ingénierie de prompts, avec une variété d'EDI (environnements de développement intégrés) d'ingénierie de prompts récemment créés. Il existe une gamme surprenante de tels outils, allant des interfaces de recherche aux EDI professionnellement conçus.
Ce chapitre fournira une vue d'ensemble de certains des outils qui pourraient vous intéresser en tant qu'ingénieur de prompts.

Consultez le dernier article de ce chapitre pour mes recommandations sur les outils à utiliser.

En raison des listes d'attente, je n'ai pas encore pu accéder à tous les EDI mentionnés ici.
De plus, en raison du nombre d'outils qui sortent, tous n'ont pas une page dédiée.

Voici un tableau des outils dont nous discuterons dans ce chapitre :
